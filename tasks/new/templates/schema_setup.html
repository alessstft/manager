В базе нет таблиц проектов и задач

Для сотрудника список проектов строится через связь «участники» — таблица new_project_members. Также нужны new_project, new_task и остальные таблицы приложения new. Обычно они создаются одной командой миграций.

Запускайте команды из папки, где лежит manage.py (manager\tasks), и тем же Python, что и для runserver.

{% if sqlite_path %}

Файл SQLite: {{ sqlite_path }}

{% endif %}

В PowerShell:

cd C:\Users\Artem\OneDrive\Desktop\str\manager\tasks
python manage.py migrate

Если пишет «No migrations to apply», а страницы всё равно падают — файл db.sqlite3 остался от старой версии проекта (например, были таблицы вроде new_post, но нет new_project_members). Единственный надёжный способ — удалить db.sqlite3 в папке с manage.py и выполнить python manage.py migrate заново (все данные в БД пропадут). Затем снова зарегистрируйте администратора и сотрудников.
